Overview

A simple interface to the Pusher REST API.

This functionality used to be part of the main PTPusher library but has been extracted into a standalone component.

The PTPusher client has support for client-sent events but if your account does not support these you can use this class to send events using the standard Pusher REST API.

As well as your Pusher API key, you will also need your app ID and secret key for signing requests. You should take care not to expose these in your application’s header files.
